# Methodology

Hullah was developed using the **Iterative Waterfall Model**, a structured software development methodology in which each phase can be revisited and refined as the project progresses.

## Development Phases

### 1. Requirements & System Analysis

- Collected and defined user needs and expectations through literature review, stakeholder surveys, and interviews with abaya designers.
- Analyzed functional requirements such as image upload, measurement extraction, and abaya recommendation.
- Identified non-functional requirements including privacy and usability.
- Determined the appropriate categories of tools and technologies for the project.

### 2. Design

- Designed the overall system architecture (frontend, backend, AI module).
- Created data flow diagrams, entity-relationship diagrams, use case diagrams, flowcharts, class diagrams, and sequence diagrams.
- Structured the user interface and built early prototypes to ensure clarity and ease of use.

### 3. System Development & Implementation

- Integrated a pre-trained pose estimation model for body keypoint detection.
- Developed the mobile frontend.
- Built the backend to handle image processing and classification requests.
- Connected the system to a cloud database for data storage and user information management.

### 4. System Testing

- Tested image processing accuracy and measurement extraction.
- Verified body shape classification logic.
- Collected user feedback to refine the user experience and validate output relevance.
- Evaluated usability, precision, and system responsiveness.

### 5. Maintenance

- Applied fixes based on testing results.
- Improved UI responsiveness and recommendation accuracy.
- Prepared the system for future updates and enhancements.

## Information Gathering

The requirements and design decisions were informed by multiple sources:

- **Literature Review** — analysis of academic research on pose estimation, body measurement extraction, and fashion recommendation systems.
- **Stakeholder Surveys** — gathering input on user needs and expectations.
- **Stakeholder Interviews** — direct interviews with professional abaya designers to translate fashion expertise (e.g., which cuts suit which body shapes) into formal classification rules.

## Project Planning

The project followed a milestone-based plan across the two terms, with early milestones (problem definition, proposed solution, data gathering, and initial design) completed in the first term, and the remaining milestones (development, testing, and evaluation) completed in the second term.
